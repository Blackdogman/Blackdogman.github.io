正则表达式

概念

  • 描述了一种字符串匹配的模式，可以用来检查一个串中是否含有某个子串、将匹配的字符串做替换或者取出符合条件的某个字符串。

    语法

  • “^” : 匹配字符串起始位置，或者文档的起始位置
  • “$” : 匹配行结尾或者文档结尾
  • “\b”: 匹配单词边界
  • “\d”: 匹配数字(0-9)
  • “\w”: 匹配数字，字母，下划线
  • “\s”: 匹配空格
  • “.” : 匹配除了换行符以外的任何字符
  • “[abc]” : 字符组，匹配括号内的字符组，一个字符，能有括号中的那些选项
    反义匹配
  • “\b” -> “\B” 不是xxx
  • [abc] -> [^abc] 不以这个匹配
    量词
  • “*”: 重复0-n次
  • “+”: 重复1-n次
  • “?”: 重复0-1次
  • “{n}”: 重复n次
  • “{n,m}”: 重复n到m次
  • “{n,}”: 重复n到更多次
懒惰匹配（匹配符合条件的最短的那条）
  • “*?”: 重复任意次，尽可能少重复
  • “+?”: 重复1-n次，尽可能少重复
  • “??”: 重复0-1次，尽可能少重复
  • “{n, m}”?: 重复n-m次，尽可能少重复
    零宽断言
  • “?=exp”: 匹配到exp前面的位置
  • “?<=exp”: 匹配到exp后面的位置
  • “?!exp”: 匹配后面跟的不是exp的位置
