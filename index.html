一、EL表达式

  • EL(Expression Language)提供了在 JSP 中简化表达式的方法，让Jsp的代码更加简化.
  • 语法：${expression}
  • el表达式的隐含对象包括：pageScope，requestScope，sessionScope，applicationScope
  • ${message} EL表达式会依次（从小到大）到pageScope，requestScope，sessionScope，applicationScope中寻找，直到找到为止。
  • 如果写成${requestScope.message}的形式，将会缩小范围只在requestScope中查找message。
  • 常见使用场景：取出request中封装的数据
    • String或基本数据类型
    • javabean
    • List<String或基本数据类型>
    • List
    • Map

事务处理

描述：保证一组数据库执行的一致性，要么都成功，要么都失败。

4大特性(ACID) 【重点】
  1. 原子性:最小的事务单元了，无法拆分，捆绑在一起，要么一起成功，要么一起失败。
  2. 一致性:数据库状态与业务逻辑是一致的，比如进行的数据交换，但是数据的总量不会发生变化。
  3. 隔离性:高并发操作中，不同事务之间要隔离开，使每个并发事务互不干扰。
  4. 持久性:一旦事务提交，事务中的数据会被持久化到数据库，即使提交事务后，数据库立马崩溃，在数据库重启时，也能通过某种方式恢复数据。
    常见问题【重点】
  5. 脏读（读取了其他事务未提交的数据）：读取到另外一个事务未提交的数据。事务2读取到事务1为提交的数据，但是事务2提交了，事务1回滚了。（读到未提交数据）
  6. 不可重复读（读取了其他事务提交了的数据）：对同一记录两次读取不一样，因为另一个事务对记录做了更改。（读到了已提交数据）
  7. 幻读（虚读）：对同一张表两次读取不一样，因为另一个事务插入了一条数据。（读到了新插入的数据）
    四大隔离级别
  8. SERIALIZABLE(串行化): 可避免脏读、不可重复读、虚读（幻读）。安全级别最高。
  9. REPEATABLE READ(可重复读) (mysql): 可避免脏读、不可重复读的发生。
  10. READ COMMITTED(读已提交数据) (oracle): 可避免脏读的发生
  11. READ UNCOMMITED(读未提交): 脏读、不可重复读、虚读都有可能发生。安全级别最低

Set接口

实现类 HashSet TreeSet

  1. 可以添加任意数据类型，HashSet可以有null，TreeSet不能有null
  2. HashSet集合无顺序，与添加顺序无关，可以存放不同类型的数据
  3. TreeSet会自动排序元素，存储同一类型数据
  4. set集合无下标
  5. 元素不可重复
  6. 常用方法

Map接口

实现类 HashMap TreeMap

  1. Map中的键值对没有顺序，与添加的顺序无关
  2. 不允许存在重复的键名，如果存在那么后添加的会覆盖前添加的
  3. 键的类型为任意类型，通常使用String类型
  4. 值的类型为任意类型
  5. HashMap中键和值都可以使用null
  6. TreeMap的key不允许为null
  7. TreeMap的键值对会被自动排序
  8. 常用方法

正则表达式

概念

  • 描述了一种字符串匹配的模式，可以用来检查一个串中是否含有某个子串、将匹配的字符串做替换或者取出符合条件的某个字符串。

    语法

  • “^” : 匹配字符串起始位置，或者文档的起始位置
  • “$” : 匹配行结尾或者文档结尾
  • “\b”: 匹配单词边界
  • “\d”: 匹配数字(0-9)
  • “\w”: 匹配数字，字母，下划线
  • “\s”: 匹配空格
  • “.” : 匹配除了换行符以外的任何字符
  • “[abc]” : 字符组，匹配括号内的字符组，一个字符，能有括号中的那些选项
    反义匹配
  • “\b” -> “\B” 不是xxx
  • [abc] -> [^abc] 不以这个匹配
    量词
  • “*”: 重复0-n次
  • “+”: 重复1-n次
  • “?”: 重复0-1次
  • “{n}”: 重复n次
  • “{n,m}”: 重复n到m次
  • “{n,}”: 重复n到更多次
懒惰匹配（匹配符合条件的最短的那条）
  • “*?”: 重复任意次，尽可能少重复
  • “+?”: 重复1-n次，尽可能少重复
  • “??”: 重复0-1次，尽可能少重复
  • “{n, m}”?: 重复n-m次，尽可能少重复
    零宽断言
  • “?=exp”: 匹配到exp前面的位置
  • “?<=exp”: 匹配到exp后面的位置
  • “?!exp”: 匹配后面跟的不是exp的位置
